隐私权政策

生效及最后更新日期：2026 年 7 月 23 日

Cookie Controller 在用户设备本地处理站点数据，不运营数据收集服务器，不包含广告、统计分析或用户跟踪功能，也不会出售或共享用户数据。

1. 适用范围

本政策适用于 Cookie Controller Chrome 扩展。该扩展的单一用途是帮助用户查看和管理其主动选择网站的 Cookie、Local Storage 和 Session Storage。搜索、复制、批量操作、导入导出、值处理工具、模板、修改历史和撤销功能均用于支持这一用途。

本政策不适用于用户通过该扩展管理的第三方网站。相关网站如何处理数据由其各自的隐私政策决定。

2. 扩展访问和处理的数据

当用户打开扩展并执行操作时，扩展可能在本地访问以下信息：

数据类别 示例 用途
标签页信息 当前窗口中的标签页 URL、标题、标签页和窗口标识 识别当前网站、列出可选网站、切换管理目标及按用户设置重新加载页面
Cookie 数据 名称、值、域名、路径、有效期、SameSite、Secure、HttpOnly 和 Cookie Store 显示、搜索、复制、新增、修改、导入、导出和删除 Cookie
网页存储数据 Local Storage 和 Session Storage 的键和值 显示、搜索、复制、新增、修改、导入、导出和删除网页存储项
用户主动保存的内容 Cookie 模板、界面偏好和自动刷新设置 在后续使用中恢复用户选择的插件配置和模板
修改记录 项目名称、域名或来源、操作类型、时间和大小变化 显示最近操作及支持当前会话中的差异查看和撤销

3. 数据处理、传输和共享

  • 仅在本地处理：Cookie 和网页存储数据在用户设备上的扩展界面与目标网页之间处理。
  • 扩展没有开发者运营的后端服务器，也不包含广告、分析、遥测或跟踪 SDK。
  • 扩展不会将站点数据、浏览信息或用户设置上传给开发者或第三方。
  • 扩展不会出售、出租或以其他方式共享用户数据。
  • 扩展不会将访问到的数据用于广告、信用评估或与站点数据管理无关的用途。
  • 当用户主动执行导出或复制操作时，所选数据会写入系统剪贴板。后续如何保存或分享该数据由用户自行决定。

4. 本地存储和保留期限

界面偏好、Cookie 模板和最近修改记录通过 chrome.storage.local 保存在用户的浏览器中，直到用户清除扩展数据、相关记录被替换，或用户卸载扩展。用于撤销操作的完整修改前后值只保留在当前弹出窗口或侧边栏会话的内存中，会在该会话结束后丢弃。

扩展读取的 Cookie、Local Storage 和 Session Storage 原始内容不会自动复制到开发者服务器。用户对网站数据所做的修改将按照浏览器和目标网站本身的规则保留。

5. 权限用途

  • cookies：读取、新增、修改、删除和监听用户所选网站的 Cookie。
  • tabs：识别、列出和切换当前窗口中的网站标签页，并在用户启用自动刷新后重新加载页面。
  • storage：在浏览器本地保存界面偏好、模板和最近修改记录。
  • scripting：在用户所选标签页运行扩展包内置函数，以管理 Local Storage 和 Session Storage。
  • sidePanel：在 Chrome 侧边栏中提供同一套站点数据管理界面。
  • HTTP/HTTPS 主机权限：允许用户在其主动选择的任意 HTTP 或 HTTPS 网站上使用 Cookie 和网页存储管理功能。

扩展不下载或执行远程代码。所有 JavaScript 均包含在提交至 Chrome Web Store 的扩展包中。

6. 用户控制和数据删除

  • 用户可以在扩展界面中清除最近修改记录。
  • 用户可以通过 Chrome 的扩展数据清理功能删除本地保存的数据。
  • 卸载 Cookie Controller 会删除该扩展的本地存储。
  • 用户可以随时停止使用扩展，或通过 Chrome 扩展管理页面禁用其网站访问权限。

7. 安全说明

Cookie 和网页存储可能包含登录状态、令牌或其他敏感信息。请只在可信设备上使用本扩展，并妥善保护复制或导出的数据。内置 JWT 工具只解码 Payload 以供查看，不验证签名，也不判断令牌是否可信或有效。

Cookie Controller 不会要求用户提交密码、付款信息或身份验证代码。请勿向不受信任的人员分享通过本扩展查看或导出的站点数据。

8. 政策变更

如果扩展的数据处理方式发生变化，本页面将同步更新，并修改页面顶部的最后更新日期。对于会显著改变数据收集或共享方式的变更，开发者会在适用情况下通过扩展说明或版本更新进行提示。

9. 联系方式

如对本隐私权政策或 Cookie Controller 的数据处理方式有疑问，请通过 GitHub 提交问题：

https://github.com/Atimynyc/cookies-manager/issues

Privacy Policy

Effective and last updated: July 23, 2026

Cookie Controller processes site data locally on the user's device. It operates no data collection server, contains no advertising, analytics, or user tracking, and does not sell or share user data.

1. Scope

This policy applies to the Cookie Controller Chrome extension. The extension has one purpose: helping users inspect and manage cookies, Local Storage, and Session Storage for websites they explicitly select. Search, copy, batch actions, import/export, value tools, templates, change history, and undo support that purpose.

This policy does not apply to third-party websites managed through the extension. Their handling of data is governed by their own privacy policies.

2. Data the extension accesses and processes

When a user opens the extension and performs an action, the extension may locally access the following information:

Data category Examples Purpose
Tab information Tab URL, title, tab identifier, and window identifier for tabs in the current window Identify the current site, list selectable sites, switch the management target, and reload a page when requested
Cookie data Name, value, domain, path, expiration, SameSite, Secure, HttpOnly, and cookie store Display, search, copy, create, update, import, export, and remove cookies
Web storage data Local Storage and Session Storage keys and values Display, search, copy, create, update, import, export, and remove web storage entries
User-saved content Cookie templates, interface preferences, and automatic reload settings Restore extension settings and templates selected by the user
Change records Item name, domain or origin, action type, time, and size change Show recent actions and support difference viewing and undo during the current session

3. Processing, transmission, and sharing

  • Local processing only: cookie and web storage data is processed between the extension interface and the target page on the user's device.
  • The extension has no developer-operated backend server and includes no advertising, analytics, telemetry, or tracking SDK.
  • The extension does not upload site data, browsing information, or user settings to the developer or any third party.
  • The extension does not sell, rent, or otherwise share user data.
  • The extension does not use accessed data for advertising, credit assessment, or purposes unrelated to site data management.
  • When a user explicitly exports or copies data, the selected data is written to the system clipboard. The user controls any later storage or sharing of that data.

4. Local storage and retention

Interface preferences, cookie templates, and recent change records are stored in the user's browser through chrome.storage.local until the user clears extension data, the relevant records are replaced, or the extension is uninstalled. Complete before-and-after values needed for undo are held only in memory for the current popup or side-panel session and are discarded when that session ends.

Cookie, Local Storage, and Session Storage content read by the extension is not automatically copied to a developer server. Changes made by the user remain according to the rules of the browser and the target website.

5. Permission use

  • cookies: read, create, update, remove, and observe cookies for a user-selected website.
  • tabs: identify, list, and switch website tabs in the current window and reload a page when automatic reload is enabled.
  • storage: save interface preferences, templates, and recent change records in local browser storage.
  • scripting: run bundled extension functions in the user-selected tab to manage Local Storage and Session Storage.
  • sidePanel: provide the same site data management interface in the Chrome side panel.
  • HTTP/HTTPS host permissions: allow users to manage cookies and web storage on any HTTP or HTTPS website they explicitly select.

The extension does not download or execute remote code. All JavaScript is included in the extension package submitted to the Chrome Web Store.

6. User control and deletion

  • Users can clear recent change records from the extension interface.
  • Users can remove locally stored data using Chrome's extension data controls.
  • Uninstalling Cookie Controller removes the extension's local storage.
  • Users can stop using the extension at any time or disable its website access through Chrome's extension management page.

7. Security notice

Cookies and web storage may contain sign-in state, tokens, or other sensitive information. Use the extension only on trusted devices and protect any copied or exported data. The built-in JWT tool decodes the payload for inspection only. It does not verify signatures or determine whether a token is trustworthy or valid.

Cookie Controller will not ask users to submit passwords, payment information, or authentication codes. Do not share site data viewed or exported through the extension with untrusted parties.

8. Changes to this policy

If the extension's data handling changes, this page will be updated and the last-updated date at the top will be revised. Material changes to data collection or sharing practices will be communicated through the extension description or a version update where appropriate.

9. Contact

For questions about this policy or Cookie Controller's data handling, open an issue on GitHub:

https://github.com/Atimynyc/cookies-manager/issues
